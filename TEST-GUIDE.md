## 测试步骤

### 1. 访问应用

开发服务器默认运行在：**http://localhost:3000**

在浏览器中打开该地址进行测试。

### 2. 测试地图加载

**预期结果：**
- 地图正常显示
- 使用 GeoQ 地图服务（深蓝紫色主题）
- 无需VPN即可访问

**检查方法：**
- 打开浏览器控制台（F12）
- 查看是否有地图加载错误
- 应该看到日志：`[DeckGLMap] Map provider: geoq`

### 3. 测试新闻源

**测试新闻RSS源：**

在浏览器中访问以下URL测试RSS代理：

```
http://localhost:3000/api/rss-proxy?url=https://www.xinhuanet.com/politics/news_politics.xml
```

**预期结果：**
- 返回XML格式的RSS数据
- 无需VPN
- 响应时间<2秒

### 4. 测试直播源

**查看直播频道列表：**
- 打开直播新闻面板
- 应该看到默认的国际频道（YouTube源）

**注意：** 中国直播源需要进一步集成到LiveNewsPanel组件中

### 5. 检查控制台日志

**正常情况下应该看到：**
```
[DeckGLMap] Map provider: geoq
[DeckGLMap] Dark style URL: /map-styles/china-geoq-dark.json
[DeckGLMap] Light style URL: /map-styles/china-geoq-light.json
```

## 常见问题

### Q: 地图无法加载？
**A:** 检查控制台是否有跨域错误，GeoQ服务应该可以直接访问

### Q: RSS代理返回403？
**A:** 域名可能不在白名单中，检查 `vite.config.ts` 的 `RSS_PROXY_ALLOWED_DOMAINS`

### Q: 直播无法播放？
**A:** 
1. 检查直播流URL是否有效
2. 检查浏览器是否支持HLS
3. 尝试使用备用直播源

### Q: 环境变量不生效？
**A:** 
1. 确保 `.env.local` 文件存在
2. 重启开发服务器
3. 清除浏览器缓存
